# UI/UX Pro Max 设计规范 - shipany-template-navigation

基于业界领先的 AI 导航站风格，本规范旨在通过高对比度、现代化动效和清晰的层级结构，提升网站的视觉冲击力与用户体验。

---

## 1. 设计系统 (Design System)

### 🎨 色彩体系 (Color Palette - "Deep Cosmos")
采用 **"Deep Cosmos"** 专业级配色，模拟顶级 SaaS (如 Linear, Vercel) 的深邃感，而非通用的灰黑配色。

| 类型 | 色值 (Hex) | 用途 |
| :--- | :--- | :--- |
| **背景主色** | `#000212` | 页面极深蓝色背景 (Deep Space) |
| **背景次色** | `#0A0B14` | 卡片背景、浮动面板 |
| **主强调色** | `#3D5AFF` | 电光蓝 (Electric Blue) - 核心操作 |
| **次强调色** | `#5E6AD2` | 靛蓝色 (Indigo) - 辅助提示、链接 |
| **文字主色** | `#FFFFFF` | 标题、高亮内容 |
| **文字次色** | `#8A8B91` | 次要描述、占位符 |
| **边框颜色** | `#1F2028` | 极细边框，配合磨砂质感 |
| **装饰发光** | `#3D5AFF22`| 只有在交互时出现的幽灵发光 |

### 🔡 字体规范 (Typography)
- **首选字体**: `Inter`, `Geist Sans` (现代科技感)
- **层级设定**:
  - **H1 (大标题)**: 36px / 2.25rem, Bold, Tracking -0.025em
  - **H2 (分类标题)**: 24px / 1.5rem, Semibold
  - **Body (正文)**: 16px / 1rem, Regular, Line-height 1.6
  - **Small (辅助)**: 14px / 0.875rem, Medium, Muted

### 📏 间距与圆角 (Spacing & Radius)
- **间距基数**: `4px` (Tailwind 比例)
- **内容边距**: 移动端 `16px (p-4)`, 桌面端 `32px (p-8)`
- **圆角规范**:
  - 卡片: `12px (rounded-xl)`
  - 按钮/输入框: `8px (rounded-lg)`
  - 徽章/小组件: `9999px (rounded-full)`

### 🌑 阴影与发光 (Shadows & Gloss)
- **卡片深度**: 使用 `1px` 的半透明边框 (`#FFFFFF10`) 代替厚重阴影，模拟超薄显示效果。
- **背景光晕**: 在关键组件后方使用极淡的 `radial-gradient` (如 `#3D5AFF05`) 提供空间深度。

---

## 2. 各板块层级结构指导 (Hierarchy Guidance)

### 🍱 布局模式: Bento Grid (核心推荐)
- **视觉重心**: 摒弃传统的等宽列表，采用 **Bento Grid** (便当盒布局)。将热门资源或每日推荐放在跨列的大卡片中，普通链接放在小卡片中。
- **动态性**: 卡片之间保持 `16px - 24px` 的较大的垂直间距，营造呼吸感。

### 🔝 导航与搜索 (Top Tier)
- **沉浸式设计**: 导航栏应采用 `backdrop-blur-md` 配合 `#000212CC`，在滚动时与背景融为一体。
- **搜索增强**: 搜索框居中，使用电光蓝 (`#3D5AFF`) 的微弱外发光作为 Focus 状态。

### 🗂 资源卡片 (Core Content)
- **高级感处理**: 隐藏不必要的边线，只在 Hover 时通过 `border-color` 切换或 `inner-shadow` 突出。

---

## 3. 组件规范 (Component Specs)

### 🔘 按钮 (Buttons - Precision Style)
- **Primary**: 纯色电光蓝背景，文字极白，无描边。Hover 时背景色小幅度变亮。
- **Secondary**: 幽灵按钮 (Ghost)，仅在 Hover 时显现文本颜色变化。

### 🃏 卡片 (Cards - Matte Glass)
- **背景**: `#0A0B1480` (带透明底)，配合 `saturate-150 backdrop-blur-xl`。
- **边框**: 上边框略亮于下边框，模拟自然光照效果。

### 🏷 徽章 (Badges - Tech Monospace)
- **风格**: 使用等宽字体 (Monospace)，字号缩小至 `12px`，全大写，间距拉开。

### 🔗 链接 (Links)
- **反馈**: 禁用下划线，改用颜色渐变过渡。

---

## 4. 深色 UI 常见陷阱 (Dark UI Pitfalls)

1.  **拒绝“死黑”背景**: 纯黑 (`#000000`) 会让页面显得廉价且在廉价屏幕上产生视觉残留。使用带蓝/紫调的深色 (`#000212`) 能增加空间深度。
2.  **避免过亮的纯白描边**: 即使是 1px 的边框，如果是 `#FFFFFF` 也会显得突兀。应使用 `#FFFFFF10` 或更低的透明度，通过“光感”而非“线条”来区分区域。
3.  **海拔 (Elevation) 与亮度正相关**: 在深色模式下，离用户越近的组件（如弹窗、卡片）背景色应该越亮，而不是通过阴影来区分。
4.  **文本对比度的“度”**: 避免对正文使用纯白。推荐使用平衡过的浅灰色 (`#D1D1D1`) 以减少长时间阅读的视觉疲劳，仅对标题使用纯白。
5.  **色彩饱和度控制**: 在深色背景下，亮色会自带“发光感”。应调低 Emphasis 颜色的饱和度，避免产生廉价的“霓虹灯”既视感。
